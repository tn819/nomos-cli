# Nomos SDK + CLI Documentation

This directory contains the auto-generated documentation site for the Nomos SDK and CLI.

## Structure

```
docs/
├── .vitepress/          # VitePress configuration
│   └── config.ts
├── guide/               # User guides
│   ├── index.md         # Introduction
│   ├── installation.md  # Setup instructions
│   ├── authentication.md
│   ├── checkout.md      # Use case guides
│   ├── invoice.md
│   └── meter-order.md
├── sdk/                 # SDK API documentation
│   ├── index.md         # SDK overview
│   ├── typedoc/         # Auto-generated TypeDoc output
│   └── ...
├── cli/                 # CLI documentation
│   ├── index.md         # CLI overview
│   └── commands.md      # Auto-generated CLI reference
└── api/                 # OpenAPI documentation
    ├── index.md         # API overview
    └── openapi.html     # Redoc interactive docs
```

## Development

### Install Dependencies

```bash
bun install
```

### Generate Documentation

Generate SDK docs (TypeDoc) and CLI docs (from Commander.js):

```bash
bun run docs:generate
```

### Start Dev Server

```bash
bun run docs:dev
```

The site will be available at `http://localhost:5173`

### Build Static Site

```bash
bun run docs:build
```

The static site will be generated in `docs/.vitepress/dist/`

### Preview Build

```bash
bun run docs:preview
```

## How It Works

### SDK Documentation

Generated by [TypeDoc](https://typedoc.org/) from TypeScript source:
- Parses JSDoc comments
- Extracts type definitions
- Generates Markdown output

Configuration: `typedoc.json`

### CLI Documentation

Extracted from [Commander.js](https://github.com/tj/commander.js/) at build time:
- Runs each command with `--help`
- Captures help output
- Generates Markdown reference

Script: `scripts/generate-cli-docs.ts`

### OpenAPI Documentation

Uses [Redoc](https://redocly.com/) to render interactive docs:
- Reads `specs/openapi.latest.json`
- Displays interactive API explorer
- Embedded as standalone HTML page

## Adding Content

### Manual Pages

Add Markdown files to `docs/guide/` and update `docs/.vitepress/config.ts` navigation.

### SDK API Changes

Update JSDoc comments in `src/*.ts`, then regenerate:

```bash
bun run docs:generate
```

### CLI Changes

Commander.js picks up changes automatically. Just regenerate:

```bash
bun run docs:generate
```

## Deployment

The built site can be deployed to any static hosting:

- **GitHub Pages**: Push `docs/.vitepress/dist/` to `gh-pages` branch
- **Vercel**: Connect repository, set output directory to `docs/.vitepress/dist`
- **Netlify**: Same as Vercel
